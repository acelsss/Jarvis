### 栈识别结论

**语言与运行时：**
- Python 3.10+（从代码风格和类型注解推断）
- 使用标准库 `logging`、`dataclasses`、`pathlib` 等

**依赖管理：**
- 包管理器：`pip`（从 `scripts/dev_bootstrap.sh` 可见）
- 当前无 `requirements.txt`，依赖管理待完善

**已有工具与目录结构：**
- **Logger**：`utils/logger.py` - 基于标准库 `logging`，提供 `get_logger()` 函数
- **Config**：`utils/config.py` - 使用 `dataclasses` 和 `os.getenv`，支持环境变量加载
- **HTTP 工具**：`utils/http.py` - 封装 `requests` 库，提供超时和错误处理
- **脚本目录**：`scripts/` - 包含 bash 脚本（`dev_bootstrap.sh`、`run_cli.sh`、`check_health.sh`）
- **配置目录**：`config/` - 包含 `.env.example` 和 systemd 服务模板

**核心模块结构：**
- `core/` - 思考与调度层（controller、intent、handlers、formatter、schemas、errors）
- `memory/` - 记忆层（client_openmemory、repo、local_cache）
- `interfaces/` - 感知与表达层（cli、voice、webhook、webcam）
- `utils/` - 工具层（logger、config、http、time_utils、security、ids、io）

**外部依赖（推断）：**
- `requests` - HTTP 客户端（从 `utils/http.py` 和 `memory/client_openmemory.py` 可见）
