## Update 2：MemoryClient 实现（自适配端点）

### 目的与要点

- 实现核心 MemoryClient，提供 `store()` 和 `search()` 方法
- 实现端点自适配机制，自动探测可用的 API 路径
- 实现指数退避重试机制（0.5s/1s/2s，最多3次）
- 实现本地待补队列，服务不可用时自动降级
- 保持代码风格一致，不引入新依赖

### 代码结构与关键函数

**新增文件：**
- `core/memory/types.py` - 数据类型定义（MemoryItem、SearchResult、make_memory_item）
- `core/memory/client.py` - MemoryClient 实现

**关键函数：**

- `MemoryClient.__init__()` - 初始化客户端，构建端点候选列表
- `MemoryClient._try_endpoints()` - 端点自适配核心逻辑，支持重试
- `MemoryClient.store()` - 存储记忆项，失败时写入本地队列
- `MemoryClient.search()` - 搜索记忆，统一映射响应格式
- `MemoryClient.flush_pending()` - 刷新待补队列，原子写入

**工具函数：**
- `utils/http.request_json()` - 新增通用 JSON 请求函数

### 端点自适配策略说明

**覆盖变量优先：**
- 若 `OM_ENDPOINT_STORE` 或 `OM_ENDPOINT_SEARCH` 已设置，直接使用该端点

**候选列表（按顺序尝试）：**
- **存储端点**：`/memories` → `/api/memories` → `/v1/memories`
- **搜索端点**：`/search` → `/api/search` → `/memories/search` → `/v1/search`

**重试策略：**
- 4xx 错误：立即失败，不重试
- 5xx 或网络错误：指数退避（0.5s → 1s → 2s），最多3次
- 某个端点成功（2xx）后立即返回

### 本地待补队列（data/pending-memories.jsonl）说明与 flush_pending 用法

**队列机制：**
- 当 `store()` 失败且超过重试上限时，自动将记忆项追加到 `data/pending-memories.jsonl`
- 文件格式：JSON Lines（每行一个 JSON 对象）
- 目录自动创建（`data/`）

**刷新队列：**
```python
from core.memory.client import MemoryClient
from utils.config import Settings

client = MemoryClient(Settings.load().om)
result = client.flush_pending()
# 返回：{"processed": 10, "succeeded": 8, "failed": 2}
```

**原子写入：**
- `flush_pending()` 使用临时文件 + 替换的方式，确保数据不丢失
- 全部成功时删除队列文件，部分失败时保留失败项
